# 0015: Migrate constants and remove TOAST_MESSAGES

## Context

`TOAST_MESSAGES` in `src/lib/constants/messages.ts` holds hardcoded strings passed to `useMutationWithToast`. These need to become `t()` calls at usage sites. Other constants with display strings also need cleanup.

## Dependencies

- 0010-0014 (components already using `t()` for most strings)

## Files

- `src/lib/constants/messages.ts` (delete or gut)
- `src/lib/constants/navigation.ts` (edit)
- `src/lib/constants/app.ts` (edit — keep non-translatable constants)
- All files that import `TOAST_MESSAGES` (edit)

## Requirements

### TOAST_MESSAGES removal

1. Find all imports of `TOAST_MESSAGES` across the codebase
2. Replace each `TOAST_MESSAGES.x.y.success` with `t('namespace:toast.x.y')` using the appropriate namespace
3. Replace each `TOAST_MESSAGES.x.y.error` similarly
4. Delete `src/lib/constants/messages.ts` (or keep only `DEFAULT_ERROR_MESSAGE` if still needed — but prefer `t('common:errors.operationFailed')`)

### Navigation constants

- `BREADCRUMB_LABELS` — either remove (use inline `t()`) or convert values to translation keys
- `HOME_BREADCRUMB.label` — use `t('common:appName')` at call site instead of constant

### App constants

- `APP_NAME` — keep as constant (used in non-translatable contexts like emails)
- `APP_TITLE` — keep for `<meta title>` tag; optionally make translatable

## Definition of Done

- [ ] `TOAST_MESSAGES` no longer imported anywhere
- [ ] `src/lib/constants/messages.ts` deleted or reduced to bare minimum
- [ ] All toast messages use `t()` at call sites
- [ ] `BREADCRUMB_LABELS` cleaned up
- [ ] No orphaned constant imports
- [ ] `pnpm check` passes

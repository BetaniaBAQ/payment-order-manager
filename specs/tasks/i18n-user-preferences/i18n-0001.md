## Context

Install `react-i18next` ecosystem and create the initialization module. This is the foundation all other i18n tasks depend on. Uses static imports (bundled JSON) to avoid flash-of-untranslated-content.

## Requirements

1. Install deps:

   ```bash
   pnpm add i18next react-i18next i18next-browser-languagedetector
   ```

2. Create `src/i18n/index.ts`:
   - Static imports for all 10 JSON locale files
   - `fallbackLng: 'es'`
   - `defaultNS: 'common'`
   - Namespaces: `common`, `orders`, `settings`, `billing`, `errors`
   - `interpolation.escapeValue: false` (React handles escaping)
   - `i18next-browser-languagedetector` with `order: ['localStorage', 'navigator']`, `lookupLocalStorage: 'i18nextLng'`
   - Export `SUPPORTED_LANGUAGES = ['es', 'en'] as const`
   - Export `type Language = (typeof SUPPORTED_LANGUAGES)[number]`
   - Export `LANGUAGE_CONFIG: Record<Language, { label: string; flag: string }>` (`es` → Espanol/ES, `en` → English/EN)

3. Placeholder JSON files: each starts as `{}` — populated in tasks 0008/0009
